# Eyes on You — 프로젝트 운영 규칙

## 세션 시작 루틴 (가장 먼저)

매 세션을 시작할 때, 본격 작업에 들어가기 전에 다음을 먼저 읽고 **"지금 어디까지 왔고 다음 후보가 무엇인지"** 방향을 사용자에게 제안한다:

1. **`docs/design/backlog.md`** — 합의됐으나 미착수인 작업 목록 (다음 작업의 단일 소스)
2. **최신 `session_logs/YYYY-MM-DD.md`** — 직전 세션의 상태·결정·미해결
3. **`docs/design/known_issues.md`** — 반복하지 말아야 할 함정/오류
4. `git log --oneline -10` + `git status` — 누적 변경

→ 이를 종합해 "직전까지 한 것 / 다음 후보 X·Y·Z / 추천 한 가지"를 한눈에 정리해 제시한 뒤 사용자 선택을 받고 시작한다. 사용자가 곧장 작업을 지시하면 그것부터 하되, 위 파일들로 맥락을 먼저 맞춘다.

## Git push 루틴

큰 작업 단위가 끝나면 **즉시 commit + push origin main**까지 진행한다. 사용자가 별도 지시 안 해도 자동.

- 매 세션 시작 시 `git status` + `git log --oneline -5`로 누적된 변경 확인. 있으면 우선 정리 제안
- 의미 있는 작업 단위(기능 추가, 시스템 변경, 텍스트 대량 갱신, 세션 로그 기록 등)가 끝나면 끊어서 commit
- commit 메시지: 한국어, `prefix(scope): 설명` 형식 (feat/fix/refactor/docs 등). 이모지 없음. 본문은 변경 항목 bullet
- main 직접 push (이 프로젝트는 branch/PR 흐름 안 씀)
- destructive 작업(force push, reset --hard 등)은 별도 확인

이 루틴이 안 지켜져서 한 세션에 31개 파일이 한 번에 누적된 적 있음(2026-05-02). 이후엔 작은 단위로 끊어서 자주 push.

## 오류 기록 루틴 (반복 방지)

세션 중 버그·설계 함정·작업 실수를 발견하면 **`docs/design/known_issues.md`** 에 "증상 → 원인 → 재발 방지책" 형태로 기록한다. 세션 종료 시 그 세션에서 나온 오류를 정리해 추가한다.

목적은 같은 실수를 두 번 겪지 않는 것 — 세션 시작 루틴에서 이 파일을 먼저 읽으므로 자연히 예방된다. 게임 버그뿐 아니라 작업 프로세스 실수(잘못된 도구 호출, 커밋 누락 등)도 포함한다.
