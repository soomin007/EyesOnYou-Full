# 본편 확장 제안 — 현재 에셋 기반

> EA/데모(현재)에서 정식 본편으로 키우는 전략. **사용자 요청(2026-06-23): "지금 에셋들을 기반으로
> 확장하려면?"** 결론 먼저: 이 게임은 **아트가 100% 절차적 코드**(CharacterArt·코드 드로잉 맵)라,
> 확장 비용이 *에셋 생성*이 아니라 **설계·코드**에 있다. 그게 최대 레버다.

## 0. 자산 현황과 제약

| 자산 | 현황 | 확장 비용 |
|---|---|---|
| **그래픽** | 전부 절차적 코드(적·플레이어·맵·HUD·이펙트) | **코드만** — 신규 적/스킬/맵/해저드/스킨 = 외부 에셋 0 |
| **BGM** | 게임플레이 5트랙(BPM 티어: main/early/mid_late/boss/hidden) + 엔딩 4 | 새 맵은 티어 재사용으로 충당. *새 "막" 테마*만 Suno 추가 고려 |
| **SFX** | 59개(ElevenLabs) | 대부분 재사용. 신규 메커니즘만 소수 추가 |
| **폰트** | Pretendard 번들 | 그대로 |

→ **병목은 BGM(곡 수)뿐.** 적·스킬·맵·해저드·스킨·스토리는 전부 코드로 늘릴 수 있다.

## 1. 피드백과의 연결 (확장 방향의 근거)

- "너무 짧다"(설문1) → **분량/런 길이** ↑
- "캐릭터 꾸미기(스킨)"(설문2 추가 희망) → **코드 스킨/코스메틱**(절차적이라 저비용)
- "더 깊은 스토리"(설문2) → **VEIL 비트·ARCTURUS·엔딩 결** 확장
- "다른 엔딩 모으러 다시"(설문2) → **다회차 차별화**(이미 착수) + 메타 진행

## 2. 확장 레버 (현재 시스템 위에서)

1. **런 구조를 막(Act)으로** — 현재 7스테이지 단선을 3막(① 침투 외곽 → ② 시설 내부/핵심부 → ③ 탈출·진실)
   으로 재편. 각 막에 고유 맵 풀·적 구성·BGM 티어(이미 early/mid_late/boss로 매핑됨). 막 사이 분량·난이도
   램프를 명시. → "짧음" 해소 + 페이싱.
2. **맵 풀 2배** — 4개 기존 템플릿(world_layout.md)에 새 레이아웃·적/해저드 배치를 더해 12 → 20+. 절차적이라
   레이아웃 좌표 + 적 dict만 추가. 글라이드 유리한 맵(backlog) 등 미충족 지형도 여기서.
3. **신규 적 2~3종** — 코드 드로잉. 후보: ⓐ **재머(jammer)** = VEIL 마커를 교란/소등(시야 붕괴 테마와 직결,
   서사적 적) ⓑ **중장(elite)** = 고HP·패턴, 막2 압박 ⓒ **부유 변형** = 드론 상위. 각각 CharacterArt + Enemy 행동.
4. **신규 스킬 / T4** — 3계열 트리 확장. backlog의 *유도 차지 사격*·*공속 라인*을 정식화, 또는 각 계열에 T4
   캡스톤. 스킨/언락과 묶어 다회차 동기.
5. **스토리 깊이** — register/degradation/??? /ARCTURUS가 이미 풍부. 막 구조에 맞춰 VEIL 비트를 더 촘촘히,
   ARCTURUS 문서·??? 단말기 풀 확장(다회차 진실 추적), 4엔딩의 분기 결을 더 두껍게(2축 구조는 유지 — 새
   엔딩보다 *기존 4개 심화*가 깔끔). 시야 붕괴 아크도 긴 캠페인에 맞춰 더 완만하게 재배치.
6. **메타 진행(영속 기반)** — 저장/도감/endings_seen/playthrough_count가 이미 영속. 위에:
   - **스킨/코스메틱**: 절차적 캐릭터 변형(팔레트·부착물 스왑), 엔딩/완주로 언락. 설문 "꾸미기" 충족, 에셋 0.
   - **언락 시스템**: 시작 로드아웃·스킬·스킨을 다회차로 개방. NG+ 결(대사 차별화 이미 착수).

## 3. 단계 제안 (의존도 순)

- **Phase A — 분량·다양성(코드만, 에셋 0):** 막 구조 재편 + 맵 풀 2배 + 신규 적 2~3 + 신규 스킬 2~3.
  난이도 곡선 재설계(difficulty_analysis 갱신). → 런이 길고 다양해짐. **가장 먼저, 가장 가성비.**
- **Phase B — 스토리 깊이:** 막별 VEIL 비트 확장 + ARCTURUS/??? 확장 + 엔딩 심화 + **인게임 대사 register/
  honorific 정식 패스**(기존 "잡아요/빠져요"→honorific, backlog 항목). → "더 깊은 스토리".
- **Phase C — 메타/코스메틱:** 코드 스킨 + 언락 시스템 + NG+ 정교화. → 리플레이·"꾸미기".
- **Phase D — 오디오 폴리시(예산 있으면):** 막별 신규 BGM(Suno) + SFX 보강. 유일한 외부 에셋 의존 단계라 마지막.

## 4. 권고

- **Phase A부터.** 에셋 0이고 피드백 1순위("짧음")를 직접 해소하며, 막 구조가 B·C의 그릇이 된다.
- 착수 시 먼저 **막 경계·맵 풀·적/스킬 목록을 설계 문서로 확정**(이 문서를 그 출발점으로) 후 코드.
- 단일 소스: 맵=world_layout.md, 스킬=growth_system.md, 스토리=STORY.md, 난이도=difficulty_analysis.md를
  확장분과 동기화.
