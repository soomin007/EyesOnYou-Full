# VEIL 대사 풀 재배치 초안 — 신뢰 단계 × 진행도 grid (검토용)

> **상태:** ✅ **구현됨**(2026-06-13). 이 grid가 `VeilDialogue.BRIEFINGS_BY_BAND`/`STORY_BRIEFINGS_BY_BAND`/
> `DEATH_BY_BAND`로 들어갔다. **구현 시 편차:** 무사망→COLD 일관성을 위해 stage 4·5·6에 **COLD 시야붕괴
> 셀을 신규 작성**(초안의 "THAW 폴백"보다 강화) + 스토리 s3 COLD 셀. 레벨업은 register 전면 분기 대신
> 실력 lead-in만(§7-5 "명료함 우선"). 나머지는 grid 그대로.

## 0. 핵심 원리 — 이건 *재작성*이 아니라 *선택 키 교체*

**현재 ACT1/2/3 풀은 이미 3개의 어투 밴드다.** 바뀌는 건 풀을 *고르는 기준*뿐:

| 밴드 | = 기존 | 어투 | 신뢰 tier(trust_score) |
|---|---|---|---|
| **COLD** | ACT1 | 작전 통신 격식 `~습니다/~하십시오`. `저도`·사과 없음 | broken/cool (0~3) |
| **THAW** | ACT2 | 격식 베이스인데 끝에 `해요`·`저도`가 샌다(균열) | neutral (4~7) |
| **WARM** | ACT3 | `해요체` 사적·걱정·사과. high는 여기에 가장 사적인 줄 + | warm/high (8+) |

**단, 한 가지 제약 — 시야 붕괴(degradation) 비트는 진행도 고정.** VEIL이 눈멀고 요원에게 시야를
넘기는 사건은 보스·???·엔딩이 막판에 일어난다고 전제한다. 그래서 **브리핑은 2차원**이 된다:

- **내용(비트)** = 진행도(stage). degradation 아크는 레일 위에 고정.
- **어투(register)** = 신뢰 단계. 같은 비트를 차갑게/따뜻하게 전달.

→ "ACT2가 너무 빨리 온다"가 해소됨: stage 2여도 신뢰가 낮으면 그 비트를 **COLD로** 전달.

## 1. 신뢰가 stage와 어긋날 수 있으니 — 도달 가능 밴드 + 폴백

trust는 0에서 천천히 오른다(추천 따름 +2 등). 그래서 stage별로 *현실적으로 닿는* 밴드만 쓰고,
벗어나면 가장 가까운 밴드로 clamp:

| stage(일반) | 주 밴드(●) / 가능(○) | 비고 |
|---|---|---|
| 0~1 | COLD ● / THAW ○ | 초반 — 거의 COLD |
| 2~3 | COLD ○ / THAW ● | 균열 비트 시작 |
| 4~5 | THAW ● / WARM ○ | 시야 새기·ACT3 진입 |
| 6 | WARM ● / THAW ○ | 역전 완성. 끝까지 차가운 플레이어는 THAW로 clamp |

**폴백 규칙:** 해당 stage에 그 밴드 셀이 없으면 → 한 단계 따뜻한/차가운 쪽 셀 사용.
(예: stage 6에 COLD 셀 안 쓰고, 끝까지 cold면 THAW로.)

## 2. 브리핑 grid — 일반 모드 7스테이지

각 stage = **비트(고정)** + 밴드별 줄. ✅=기존 줄 재활용, ✎=신규/격식화 필요.
줄은 풀이라 같은 셀에 여러 개 — 회차 다양성 유지.

### stage 0 — 첫 진입(경비 느슨, 안심) · 비트: 도입
- **COLD ●** ✎ "경비가 느슨합니다. 전방 경로가 더 빠릅니다." / ✎ "첫 임무입니다, 요원. 지원하겠습니다." / ✎ "여기서 감각을 익히십시오. 무리하지 마십시오."
- **THAW ○** ✅ "경비가 느슨한 편이에요. 앞쪽이 더 빨라요." / ✅ "첫 임무예요, 요원. 제가 봐줄게요."

### stage 1 — 진입 성공, 통로 좁아짐 · 비트: 전방 안내
- **COLD ●** ✎ "내부에 진입했습니다. 경비 패턴을 확인했습니다." / ✎ "두 번째 구역입니다. 잘 따라오고 있습니다." / ✎ "여기서부터 통로가 좁습니다. 전방은 제가 확인하겠습니다." *(§7-2)*
- **THAW ○** ✅ "안으로 들어왔어요. 경비 패턴 보여요." / ✅ "두 번째예요. 잘 따라오고 있어요."

### stage 2 — 흠칫(degradation 전조) · 비트: 앞의 뭔가에 멈칫→얼버무림
- **COLD ○** ✎ "전방에... 잠시. 아닙니다. 경로 유지하십시오." / ✎ "방금 무언가. ...오인입니다. 진행하십시오."
- **THAW ●** ✅ "저 문 너머는... 잠깐. 아니에요. 가던 길 가요." / ✅ "앞에 뭔가... 아니에요. 제가 잘못 봤어요. 가요." / ✅ "방금... 아니에요. 신경 쓰지 말아요. 저도 가끔 이래요."

### stage 3 — 봉인/오래된 구역 · 비트: 도면 불일치(저도 처음)
- **COLD ○** ✎ "이 층은 도면과 다릅니다. 기록에 없는 구역입니다." / ✎ "어딘가 잠긴 문이 있습니다. 봉인 주체 불명."
- **THAW ●** ✅ "이 층은 도면이랑 달라요. 저도 처음 봐요." / ✅ "어딘가 잠긴 문이 있을 거예요. 누가 봉인했는진 저도 몰라요." / ✅ "이 구역은 오래됐어요. 오래 닫혀 있었고요."

### stage 4 — 시야 새기 시작(degradation 1단계) · 비트: 일부를 요원에게 넘김 **[고정]**
- **THAW ●** ✅ "여기부터는 잘 안 보여요. 이쪽은 요원이 봐줘요." *(머리위→이쪽, §7-4)* / ✅ "자꾸... 놓쳐요. 이런 적 없는데. 요원이 직접 봐요."
- **WARM ○** ✅ "제가 못 보는 데가 생겨요. 거긴 요원이 봐줘요."
- **COLD(폴백)** ✎ "이 구간 시야 확보가 어렵습니다. 전방은 요원이 직접 확인하십시오." *(차가운 플레이어용 — 비트는 유지)*

### stage 5 — ACT3 진입·서버 근접·??? 복선 · 비트: 임무 외 말
- **THAW ○** ✎ "서버실이 저 아래예요. 천천히 가도 됩니다." / ✎ "도면에 없는 길이 하나 있어요. 저도 잘 모르겠어요."
- **WARM ●** ✅ "서버실이 저 아래예요. ...요원. 천천히 가도 돼요." / ✅ "여기 도면에 없는 길이 하나 있어요. 저도 잘 모르겠어요." / ✅ "요원. 끝까지 따라와줘서, 고마워요."

### stage 6 — 역전 완성(degradation 완성) · 비트: VEIL 거의 못 봄→요원이 대신 봄 **[고정]**
- **WARM ●** ✅ "다 와서... 앞이 안 보여요. 이런 적 없어요." / ✅ "여기, 제가 잘 안 보여요. 이제 요원이 봐줘요. 저 대신." / ✅ "지금부터 제가 틀릴 수도 있어요. 미리 말해둬요."
- **THAW(폴백)** ✎ "여기, 시야가 거의 끊겼어요. 이제 요원이 봐요. 저 대신." *(차가운 플레이어도 역전 비트는 받음)*

> **신규(✎) 분량:** stage0~3 COLD 셀 + stage4~6 폴백 한 줄씩. 대부분 `rewrite.md` ACT1 격식 제안과
> §7에 이미 있음 → 거의 옮겨담기.

## 3. 브리핑 grid — 스토리 모드 5스테이지

스토리는 곡선이 짧아 s2가 흠칫+시야새기를 겸한다. degradation 비트: s3 보스에서 역전.

| stage | 비트 | COLD | THAW | WARM |
|---|---|---|---|---|
| 0 | 도입 | ✎ "경계가 느슨합니다. 여기서 감각을 익히십시오." ● | ✅ "경계가 느슨해요. 여기서 감을 잡아요." | — |
| 1 | 진입 | ✎ "내부에 진입했습니다. 경비를 확인했습니다." ● | ✅ "안으로 들어왔어요. 경비 보여요." | — |
| 2 | 흠칫+시야새기 **[고정]** | ✎ "전방에... 아닙니다. 경로 유지." / "이쪽 시야가 어렵습니다. 요원이 확인하십시오." | ✅ "저 문 너머는... 잠깐. 아니에요. 가던 길 가요." ● / ✅ "여기부터는 시야가 흐려져요. **이쪽은** 요원이 봐줘요." *(머리위 오류 수정 §7-4)* | ○ |
| 3 | 역전 완성 **[고정]** | ✎(폴백) "시야가 거의 끊겼어요. 이제 요원이 봐요." | ○ | ✅ "다 와서 앞이 안 보여요. ...요원이 봐줘요." ● / ✅ "여기서부터 잘 안 보여요. 이제 요원 차례예요. 저 대신." |
| 4 | 탈출 | ✎ "확보했습니다. 탈출 경로로." | ✅ "잡았어요. 이제 빠져나가요." ● | ✅ "조용히 빠져요. 거의 다 왔어요." |

> **머리 위 오류:** 스토리 모드는 드론 배제 → s2의 "머리 위"를 "이쪽"으로 일반화(전 밴드 공통).

## 4. 사망 메시지 — 신뢰 밴드 × 맥락 + 실력 오버레이

사망도 **ACT→신뢰밴드 재키**. 맥락(첫/추천따름/추천무시)은 유지. 실력(competence)은 *오버레이*로
얹는다(grid 안 늘림).

| 밴드 | 첫 사망(gentle) | 추천 따랐는데 죽음(VEIL 책임) | 추천 무시하고 죽음 |
|---|---|---|---|
| **COLD** | ✎ "첫 손실입니다. 재정비하고 다시 갑니다." / "괜찮습니다, 요원. 처음입니다." | ✎ "제 경로가 까다로웠습니다. 다시 갑니다." / "조언이 적절치 않았습니까. 재시도하십시오." | ✎ "다른 경로를 택하셨군요. 다시 갑니다." / "이 경로는 맞지 않았던 것 같습니다." |
| **THAW** | ✅ "저도 좀 걱정됐어요. 다시 가요." / "이 구역이 어려워요. 같이 풀어봐요." | ✅ "제 말을 믿었는데 결과가 좋지 않았어요. 미안해요." / "제 판단이 틀렸어요. 미안해요, 요원." | ✅ "제 말은 안 들었는데, 결과는 비슷했네요." / "요원 방식대로 해봤는데 쉽지 않죠." |
| **WARM** | ✅ "거의 다 왔어요. 다시 해요." / "여기서 멈추지 않아도 돼요, 요원." | ✅ "제가 잘 못 봐서... 미안해요. 다시 가요." / "마지막인데 쉽지 않네요. 저도요." | ✅ "여기서 멈추지 않아도 돼요, 요원." / "요원 방식이 틀린 건 아니었어요." |

**실력 오버레이 (§5):**
- **struggling**(사망 누적·death≥3) → 위로 풀 추가: ✅ "이 임무가 너무 힘든 거면 말해줘도 돼요." /
  "제가 더 잘 안내했어야 했어요." (WARM) · COLD/THAW도 한 톤 더 부드럽게.
- **skilled**(드물게 죽은 고수) → terse + 의외라는 톤: ✎ "흔치 않네요, 요원. 바로 갑니다."(COLD) /
  "여기서 잡힐 줄은 몰랐어요. 다시 가요."(WARM) [alt: "이런 데서 멈출 요원이 아닌데. 다시 가요."]

## 6. 코드 형태 미리보기 (참고 — 반영 시)

```
# 풀을 [밴드][stage] 2차원으로. 밴드=trust tier, stage=비트.
# get_briefing: 밴드 = trust→band, stage→비트 행. 해당 셀 없으면 폴백.
static func get_briefing(stage_index: int) -> String:
    var band: String = GameState.veil_register_band()   # "cold"/"thaw"/"warm"
    var pools: Dictionary = STORY_BRIEFINGS if GameState.story_mode else BRIEFINGS
    var cell: Array = _resolve_cell(pools, band, stage_index)  # 폴백 포함
    return _pick(cell)
```
- `veil_register_band()`는 §3.4 임계로 trust_score→3밴드.
- 사망: `get_death_briefing(band, context, competence)`로 시그니처 확장.
