# 막(Act) 정체성 — 설계 단일 소스

> **목적**: 플레이테스트 평결("막이 플레이로 안 느껴짐 = 스테이지 번호 라벨일 뿐", 2026-06-25)에 답한다.
> 막을 *느껴지게* 만드는 **재사용 틀**을 박고, 그 틀로 막1을 먼저 완성해 "이게 막이다"를 증명한다.
> 이 문서가 막 정의의 단일 소스 — 새 막/맵을 추가할 땐 이 틀을 채워야 한다(reskin 재발 방지 계약).
>
> 단일 소스 연동: 막 데이터 = `GameState.ACTS`, 맵 = `RouteData.ALL_ROUTES`, 스토리 = `STORY.md`.

---

## 0. 현재 상태 (왜 막이 안 느껴지나)

막은 `GameState.ACTS`에 데이터로 존재한다 — `{id, name, bgm, stages, boss}`. 하지만 **플레이어에게 막은
한 번도 노출되지 않는다.**

- Briefing은 `STAGE N / 9`만 표시. "ACT I · 침투" 막 진입 카드도, 막을 넘는 전환 비트도 **없음**.
- 막을 실제로 가르는 차이는 ① 등장 맵 풀(`min/max_stage` 범위) ② BGM 티어 — **둘뿐.**
- 막1 맵 8종은 공유 정체성이 없다. risk가 1~3 뒤섞이고(watchtower risk3가 오프닝 s1에 등장),
  "외곽 침투"라는 한 아이디어로 묶이는 큐레이션이 없다 → 8개의 비슷한 가로 통로로 읽힌다.

→ **막 = 데이터일 뿐 경험이 아니다.** 이 문서가 막을 *경험*으로 끌어올린다.

---

## 1. 재사용 틀 — "막의 정체성" 5요소 (anti-reskin 계약)

새 막을 만들거나 맵을 추가할 때, 아래 5칸을 **반드시** 채운다. 한 칸이라도 "다른 막과 같다"면 그건 막이
아니라 스테이지 묶음이다. 맵은 *자기 막의 성격·팔레트에 복무*해야 풀에 든다 — 이름만 바꾼 통로는 거부.

| # | 요소 | 묻는 것 |
|---|---|---|
| 1 | **성격(Character)** | 이 막에서 플레이어는 *무엇을 하는 사람*인가? (한 문장 판타지) |
| 2 | **적·해저드 팔레트** | 이 막에만 허용/금지되는 적·해저드. (막 경계 = 새 위협의 도입선) |
| 3 | **난이도 램프** | 막 안에서 압박이 어떻게 오르는가. 막 첫 stage = 진입, 끝 stage = 고조. |
| 4 | **전환 비트(Entry)** | 막에 *들어서는 순간*의 연출. 카드 + VEIL 톤 + BGM 전환 + 첫 새 위협. |
| 5 | **대비 좌표(Contrast)** | 이 막은 전체 곡선의 어디인가 — 익숙함/낯섦, 데모 영역/라이벌 영역. |

> **대비 원칙(백로그 제약)**: 막1~3 언저리는 *데모 느낌 유지*(데모 유저 익숙함). *낯선/새로운* 체감은
> 막4~5(라이벌 VEIL 영역)에서 터뜨린다. 그러므로 **초반 막의 "정체성"은 새 기믹이 아니라 — 톤의 일관성 +
> 절제 + 전환 비트 + 난이도 램프**다. 기믹 폭발은 후반의 몫. 막1의 증명 목표는 "익숙한데도 막이 *느껴진다*".

## 2. 막1 — 침투 (INFILTRATION) · stage 0~2

### 1) 성격
**당신은 어둠 속의 침입자다.** SILO-7 외벽·외곽을 따라 경비망 사각으로 스며든다. 정면 교전이 아니라
*지나가는* 막 — VEIL이 "다 싸울 필요 없어요"라고 안내하는, 게임의 동사(이동·점프·대시·선택)를 익히는 도입부.
판타지 한 줄: **"불 꺼진 바깥을, 들키지 않고, 안으로."**

### 2) 적·해저드 팔레트
막1은 **지상 인간 경비만.** 시설 내부 기술 위협(드론·증기·포탑·레이저)은 전부 막2 이후로 **금지** —
막2 진입선이 "안으로 들어가니 기계가 깨어난다"로 읽히게 하는 게 핵심.

- **허용**: patrol(순찰), 소수 방패병(근접 차단 1), 가벼운 저격(노출 지형 압박용, 십자포화 금지).
- **금지(막2+ 전용)**: 드론, 증기 분출구, 발사 포탑/트랩, 검문 레이저, 저격 둥지 십자포화, 폭격기.
- **해저드 = 어둠·노출뿐.** 위협은 *지형*(엄폐 부재·높이·갭)으로 준다. 장치가 아니라 동선이 압박.

### 3) 난이도 램프 (현재 risk 뒤섞임 → 정렬)
- **s0 — 진입**: 검증된 안전 오프너만(risk 1~2). 전투 최소, "이렇게 움직이는 거구나". 현재 back_alley/
  rooftops 2종 → 유지. **방패병·저격 맵은 s0 금지**(이미 parking_lot/perimeter 제외 적용됨).
- **s1 — 변형**: 첫 가벼운 압박 도입(방패병 1, 또는 가벼운 노출). risk 2 중심.
- **s2 — 고조**: 막1 안에서 가장 험한 길(노출+저격, risk 2~3). 막2 진입 직전 긴장 끌어올림.
- **정렬 과제**: watchtower(risk3)를 s1에서 빼 s2 전용으로(또는 막2 초입으로 이동) — 오프닝 막의 난이도
  outlier 제거. 막1 풀은 "어둠 엄폐형(우회)"과 "노출형(저격 가벼이)"의 두 결로만 큐레이션.

### 4) 전환 비트 (막1 진입 / 막1→막2)
- **막1 진입(s0)**: 기존 OPERATION PALIMPSEST 인트로 **앞**에 `ACT I · 침투` 타이틀 카드 1초.
  VEIL 톤 = COLD(딱딱한 지시). BGM = early 티어.
- **막1→막2 진입(s3)**: 이게 "막이 느껴진다"의 핵심 증거.
  `ACT II · 잠입` 카드 + VEIL 한 마디("안으로 들어가요. 여기부턴 기계가 봐요.") + BGM early→mid_late +
  **첫 드론 등장**. 바깥(인간 경비) → 안(기계 감시)의 *문턱*을 4중(카드·말·음악·새 적)으로 동시에 친다.

### 5) 대비 좌표
**곡선의 시작 = 가장 익숙한 자리.** 막1은 데모 유저가 "아 이거지" 하는 베이스라인. 여기서 새 기믹을
쓰지 않는 게 의도 — 막4~5에서 라이벌 VEIL이 이 익숙함을 *배신*할 때 대비가 살아난다. 막1의 임무는
판타지·동사·VEIL 관계의 *기준선*을 또렷이 세우는 것.

## 3. 막1 증명을 위한 구현 목록 (설계 승인 후 착수)

> 작게 증명 → 키우기. 아래는 막1 한 막에 한정한 최소 구현. 검증되면 막2·3에 같은 틀로 확장.

- **B-1 막 진입 카드**(최대 레버, 최소 위험): `act_for_stage`가 바뀌는 stage(0/3/6) 진입 시 Briefing 앞에
  `ACT N · 이름` 카드. 막 안 stage엔 안 뜸. `GameState.ACTS`의 name 사용 — 데이터 단일 소스.
- **B-2 막1 팔레트 강제**: 막1 맵 풀에서 드론/함정 적 정의 제거 확인 + 막1 맵은 인간 경비만. (대부분
  이미 그러함 — 누수 점검 + 주석으로 계약 명시.)
- **B-3 난이도 램프 정렬**: 막1 맵의 등장 stage를 램프(s0 안전 / s1 변형 / s2 고조)에 맞춰 재배치.
  watchtower s1→s2. 막1 맵을 "우회결/노출결" 두 갈래로 정리.
- **B-4 막1→막2 문턱 연출**: s3 진입 전환 비트(카드 외 VEIL 한 마디 + 첫 드론을 *연출적으로* 첫 등장).

각 단위 후 실제 플레이(막1 완주 → 막2 진입)로 "막이 느껴지는가"를 사용자와 검증. reskin 재발 방지의
실증은 **막1을 통과하는 체감**으로만 판정한다.

## 4. 막2·막3 (막1 검증 후 — 칸만 예약)

같은 5요소 틀로 채운다. 초안 좌표만:

- **막2 · 잠입(s3~5)**: 성격 = "기계의 눈 아래 시설 내부를 가로지른다". 팔레트 = 드론·증기·포탑·검문
  레이저 *도입*. 램프 = 함정·드론 밀도 상승. 대비 = 여전히 데모 영역이나 위협이 *장치화*된다.
- **막3 · 진실·탈출(s6~8)**: 성격 = "핵심부로, 진실로". 팔레트 = 복합(전투 풀)+보스(lab)+정적(???).
  여기서 라이벌 VEIL 첫 간섭(`rival_veil_concept.md`)의 *씨앗*. 대비 = 데모↔라이벌 영역의 경첩.
