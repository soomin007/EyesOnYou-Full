# 데모 플레이테스트 피드백 분석 (2026-06-26)

> 데모 버전에 대한 상세 피드백 1건 도착(두서없는 느낀점 모음). 코드 대조 + backlog/known_issues 교차로
> "이미 해결 / 진짜 빈틈 / 의도된 동작 / 오해"를 갈라 수용 판정. 대부분 데모·본편 공통 적용.
>
> **핵심 진단**: 약 80%가 *명확성/온보딩* 한 줄기 — 시스템은 있는데 *안 보임*. 테스터 본인이 정확히 짚음:
> **"사람들은 생각보다 텍스트를 주의깊게 안 읽는다."** → 시각 피드백 보강이 최고 가성비.

## 코드 대조로 확인한 사실
- **위협 화살표 각도**: 이미 `dir = (적 - 플레이어).normalized()` 동적 계산(VeilSight `_draw_edge_arrow`).
  "고정되어 어색"은 구 빌드/오해. 단 `EDGE_MARGIN=48`이라 가장자리에서 떨어진 건 사실(줄일 여지).
- **레버**: `LeverInteractable`은 점멸 빛(hint glow)만, **텍스트 프롬프트 0** → 피드백 유효.
- **점프 바인딩**: W·Z·Space·**Up** 모두 jump(현 코드). 코요테 타임/점프 버퍼 구현됨(`_coyote_t`/`_jump_buffer_t`).
- **엔딩 막판 선택**: 데모는 구 추천수용 2축. **본편(B2/B3)은 처리 선택 4종이 엔딩 결정** → 본편서 이미 해결.

## 🟡 Tier 2 — 밸런스/디자인 판단 필요 (수용하되 사용자 결정)

- **쉴드 사기**(아무데서나 회복→깔짝) — backlog shield T3 재충전과 연결. 재충전 조건/쿨 재검토.
- **글라이드 강제·느린 낙하·터렛 회피 방해, 홀드형(스페이스 홀드=글라이드) 선호** — 부분 유효. 홀드형은 큰
  조작 변경, 강제성/낙하속도는 튜닝 여지. backlog 글라이드 이력과 함께.
- **더블점프 애매한 높이차**(그냥점프 불가·더블도 타이밍 빡빡) — 발판 간격을 제자리 더블점프 정점에 맞춰
  검증(known_issues 동형 경고 존재).
- **머리높이 발판에 내 총알 막힘**(코앞 사격) — 얇은/원웨이 발판 탄 통과 고려. 의도 확인 필요.
- **대시 끝까지 안 씀**("2회차에 이동기로 쓰니 쾌적") — 온보딩/가치 강제 구간. 기존 backlog(대시 역할)와 연결.

## ⚪ Tier 3 — 이미 해결/의도/사소

- **막판 스테이지 선택이 엔딩에 영향 없음** — ✅ 본편 B2/B3(처리 선택)로 해결. 데모 한계가 맞음.
- **공중 점프 가능(걸어 떨어지며)** — ⚪ 코요테 타임(의도된 손맛 기능, 구현됨).
- **위키 누르면 점프 안 됨** — ⚪ 현 코드 Up=jump 바인딩됨(데모 빌드 cfg override 가능성).
- **화살표 각도 고정** — ⚪ 현 코드 동적(char→적). margin/가독성만 유효(T1-9).
- **K 대시 어색(Ctrl 선호)** — ⚪ 키 재설정 가능. 사소.
- **튜토 첫 점프 빡빡·더블점프 안 써도 통과** — 🔶 유효(소). 쉬운/더블필수 코스 분리 아이디어 좋음.
- **튜토부터 전체 스킬트리=벽** — ✅ 유효(온보딩). 초반 일부만 노출, 점진 공개. (튜토 개편 = 별 작업단위)
