## 0. 방향 전환 (2026-06-23) — 부스 종료 → 웹 개인 플레이 / EA 데모 → 본편

> **과제전 부스 전시는 끝났다.** 이제 설계 기준은 **"웹 환경에서 각자(개인) 진행"**. 부스/QR/키오스크
> 가정(매 타이틀 진입=새 세션, 도감 리셋 등)은 더는 기준이 아니다. **현재 빌드 = 얼리 액세스/데모 느낌으로
> 마감**(다회차·리플레이 포함), **그다음 본편을 실제 게임처럼 분량 확 늘려 제작**.

- **본편 확장(분량 대폭 ↑)**: 스테이지/맵/적/스토리/엔딩을 정식판 규모로 확대. 현재 = EA 데모. *다음 큰 단위.*
  → **제안/전략 단일 소스 = [`expansion_plan.md`](expansion_plan.md)** (아트가 절차적 코드라 확장은 코드·설계,
  병목은 BGM뿐. Phase A 분량·다양성[에셋 0] → B 스토리 깊이 → C 메타/스킨 → D 오디오). 착수 시 막 경계·맵 풀·
  적/스킬 목록부터 확정.
- **웹 개인 플레이 전환**: (1차 ✅ 적용) 부스 도감 리셋(`Title._ready` `seen_enemies.clear()`) 제거 →
  도감·본 엔딩(`endings_seen`)·완주 카운트(`playthrough_count`) 영속(settings.cfg). **단일 자동저장
  이어하기** — `user://run.cfg`에 RouteMap 진입마다 자동저장, 타이틀 "이어하기"로 RouteMap 복귀
  (`save_run`/`load_run`/`has_run`/`clear_run`). 엔딩 도달 시 메타 적립 + run 삭제, 새 게임 시 run 삭제.
  **실측 현황**: 영속은 **GitHub Pages(github.io)에서 사용자 확인 완료**(브라우저 완전 종료 후 스킬·레벨·
  도감 유지). 메인 호스트 = github.io. 웹에선 종료 버튼 숨김(브라우저가 탭을 못 닫음). **남은 점검**: 웹 전용
  이슈(오디오 autoplay 정책·입력·캔버스 크기·성능) 정도. 다회차 리플레이 대사 차별화(아래)는 별도 —
  이제 `playthrough_count`/`endings_seen`이 신호로 준비됨.
- **VEIL 시야 붕괴 onset**: (데모 ✅ 적용) 보스/탈출 직전 첫 전투 맵(일반 stage>=4 / 스토리 stage2~3)을
  **"진입부터 붕괴"**로 — `record_route_choice`가 onset에서 `veil_degraded`+`veil_reversal_pending`을 켜
  VeilSight가 시작부터 어둡고, 진입 시 역전 멘트 1회(`Stage._setup_veil_mistakes`). 중간 글리치·자막 겹침
  ("갑자기 와다닥") 제거(사용자 보고). 보스 ARENA·탈출은 상속/제외. **본편 과제(남음)**: 확장 스테이지
  구조에 맞춰 역전 비트 위치·강도 정식 재설계(현재 stage>=4 첫 발동은 데모용 임시 정렬).
- **인게임 대사 register/붕괴 인식**(본편 대사 패스, 사용자 보고 2026-06-23): `register_band`(딱딱→따뜻)와
  시야 붕괴 상태가 **브리핑·사망·위협콜아웃에만** 반영되고, **entry_comment·보스 경고·ACT3 자막·스킬 멘트는
  해요체 하드코딩**이라 초반 COLD 구간·붕괴 상황과 톤이 어긋남(예: 보스전에서 "못 봐요" 상태인데 경고는
  자신만만한 해요체 전술 콜아웃). 인게임 자막을 밴드/붕괴 인식하게 정식 재설계(대사 다수 재작성 + COLD
  변형 작성). 어투 결정은 사용자 영역 — 본편 작업 단위로.

## A. 플레이테스트로만 확인 가능 (내가 못 함 — 사용자 플레이 필요)

- **피드백 설문 응답 수집** → 1차 응답 3건 도착(06-19·06-23 설문 + 교사반 카톡). 명확성(최종목적·보스안내·
  회피정당화) + 보스/드론 완화 반영 완료(2026-06-23). 추가 응답으로 체감 튜닝 지속. ([[feedback-form]])
- **내보낸 빌드 체감**: 해상도 실제 적용·텍스트 선명도·비네트 농도(0.10)·맵 간 어두움 지속. export 설정 도움 가능.
- **VEIL 추천 임계값 튜닝**: `PERF_SKILLED_HITS_MAX=1` / `PERF_STRUGGLE_HITS_MIN=4`(최근 2스테이지 평균)가 체감과 맞는지.
- **신뢰 단계 튜닝**: trust 임계(0/4/8/12), 수용률 0.5, 게이지 스케일[2,4,8,12,16], 취약함 게이트 체감(무사망→COLD가 너무 차갑진 않은지).
- **스킬 외형 점검**: 전투 중 총기 회전·부착물 겹침·가독성, 획득/티어업 플래시 체감, 활강 유도↔사격강화 관통 분리 체감.
- **함정/저격수 난이도**: 후반 트랩(datacenter·watchtower), 둥지 저격수 LoS·사선 노출, 글라이드 게이트 도달 난이도.
- **shield T3 재충전**: 재충전 시간(30s)·재무장 플래시 톤.
- **포스터 2배 고해상 인쇄 가독성** + 필요 시 PNG→PDF 외부 변환.

## C. 코드 정리 (착수 시 플레이테스트 동반)

- **Stage.gd 분리** — 3580줄 god-object, 수십 개 공유 멤버로 엮임. 자율 보류. 단계적 안전 계획:
