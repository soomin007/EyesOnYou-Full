# STORY_REDESIGN v2 — 플레이테스트 회신 (클로드 코드 → 스토리 작가)

> 작성: 2026-06-07, 클로드 코드
> 맥락: v2 텍스트(인트로 재작성 · 세 VEIL "봤다" triad · 두려움 에스컬레이션 브리핑)를
> 게임에 반영하고, 사용자가 **스토리 모드**로 직접 플레이테스트함. 그 결과를 돌려보냄.
> 보이스 규칙은 그대로 유지: 40자 내외 / "요원" 호칭 / em dash 없음 / "저도"는 ACT2부터.

---

## 0. 한 줄 요약

**두려움 에스컬레이션이 플레이어에게 전혀 전달되지 않았다.** 사용자 원문:
> "두려움 뭐시기를 어디서 느껴야 하는지 모르겠어. 전이랑 뭐가 달라졌지?"

텍스트 자체는 반영됐지만 **구조 + 노출 빈도 + 워딩** 세 겹의 문제로 비트가 새어나간다.
아래가 진단이고, 이건 워딩만의 문제가 아니라 **전달 채널 설계** 문제이므로 작가/코드 양쪽이 필요하다.

---

## 1. 왜 안 느껴졌나 — 진단 (가장 중요)

### 1-1. 브리핑이 stage당 풀에서 "랜덤 1줄"이라 비트가 희석됨 ★핵심
현재 `STORY_BRIEFINGS` stage 3(=보스 직전, 두려움 최고조여야 할 자리):
```
[
  "이상하죠. 다 와서 더 안 보여요.",      ← 두려움 비트
  "끝까지 같이 가요. 제가 보는 한.",       ← 안심/동행 톤
]
```
- stage 진입 시 **둘 중 하나를 무작위로** 뽑는다. 즉 **50% 확률로 두려움 줄을 아예 못 본다.**
- 사용자는 이번 판에 안심 줄을 뽑았을 가능성이 높고, 그래서 "전이랑 뭐가 달라졌는지 모르겠다"가 된다.
- v2 설계 의도는 "어느 줄이 뽑혀도 그 stage의 비트를 운반"이었지만, **실제 풀은 비트 줄과 비-비트 줄이 섞여 있어** 그 의도가 깨졌다.

### 1-2. 단일 화면 1회 노출 — 놓치기 쉬움
- 브리핑은 stage당 **딱 한 번, 한 화면**. 사용자는 브리핑을 ENTER로 빠르게 넘기는 습관이 있음(이전 보고).
- 두려움 아크처럼 "누적"이 핵심인 연출을, 놓치기 쉬운 단일 채널에만 실으면 누적이 성립 안 함.

### 1-3. 워딩이 "두려움"이 아니라 "전술 코멘트"로 읽힘
- `"이상하죠. 다 와서 더 안 보여요."` → 플레이어에겐 *시야가 막혔다는 정보 안내*로 읽힌다. VEIL이 **무서워한다**는 신호가 없다.
- `"여기서부터 제 시야가 늦어요."`(일반 모드 stage 4)도 마찬가지 — 담담한 보고체라 정서가 안 실린다.
- "안 보인다"가 공포가 되려면, **안 보이는 것 자체를 VEIL이 두려워한다**는 게 표면에 드러나야 한다.

### 1-4. 스토리 모드는 5스테이지라 곡선이 짧다
- 두려움 누적이 stage 2(첫 멈칫) → 3(최고조) 단 두 비트로 끝난다. 빌드업 여유가 없음.
- 즉 스토리 모드에서는 비트가 **더 진하고 확실**해야 한다 (희석되면 곧장 0).

## 2. 내가 이번 빌드에서 이미 처리한 것 (작가 작업 아님)

| 항목 | 처리 |
|---|---|
| **인트로 VEIL 말투** | 단문 파편 `"저는 앞을 봐요. 요원은 눈앞을."` → 완결 문장 `"멀리는 제가 봐줄 테니, 눈앞은 요원이 맡아줘요."`로. 시야 분담 모티프는 유지, 어색함만 제거. (사용자: "말투가 너무 이상하다") |
| **맵 카드 중복** | 카드에서 `description`과 `veil_comment`가 같은 위협을 두 번 서술하던 것 → **VEIL 코멘트 한 목소리만** 노출. `description`은 이제 카드에 안 뜬다. (사용자: "설명 부분과 밑 VEIL 코멘트가 너무 겹친다") → **§3-D 재활용 결정 필요** |
| **엔딩 C 게이팅** | 이스터에그 미방문 시 "물어볼 거 있다/없다" 선택지 제거. 맥락 없이 분기되어 갑작스럽던 문제 해소. |

## 3. 작가에게 부탁 (다음 버전 v3)

### A. 두려움 풀을 "비트 동질화" — 어느 줄이 뽑혀도 무섭게 ★최우선
구조(랜덤 선택)는 그대로 두되, **두려움 stage의 모든 줄이 같은 비트를 운반**하도록 다시 써줘.
즉 stage 3 풀의 2~3줄 **전부**가 "VEIL이 안 보이는 걸 두려워한다"여야 한다. 안심 줄을 같은 풀에 섞지 말 것.

예 (느낌 참고용, 그대로 쓰지 말고 더 좋게):
```
stage 3 (보스 직전):
  "다 와서… 앞이 안 보여요. 이런 적 없었는데."
  "여기, 제 눈이 안 닿아요. 요원, 제가 못 보면 요원이 봐줘요."
  "무서운 말 같지만… 지금부터 제가 틀릴 수 있어요."
```
- "끝까지 같이 가요. 제가 보는 한." 같은 **안심 줄은 별도 채널**(예: 사망 후 멘트나 보스 클리어 직후)로 옮기는 게 좋겠다.

### B. "안 보인다 → 무섭다" 전환을 표면화
- 보고체("시야가 늦어요")가 아니라 **VEIL의 동요**가 드러나게: 말 끊김, 머뭇, "이런 적 없어요", "왜 이러지" 같은 자기 당황.
- em dash는 못 쓰니 `…`(말줄임)이나 짧은 끊긴 문장으로 호흡을 만들어줘.

### C. 두려움을 단일 브리핑 말고 분산할지 결정
비트가 놓치기 쉬운 건 **단일 채널 의존** 탓이 크다. 후보 채널:
- `entry_comment` (맵 진입 직후 인게임 자막 — 브리핑보다 놓치기 어려움)
- 사망 멘트 (ACT3에서 이미 개인적 톤)
- 보스전 중간 VEIL 한마디
무엇을 쓸지 정해주면 **연결은 내가 한다.** (지금은 브리핑에만 있음)

### D. 맵 `description` 역할 재정의 (중복 원천 제거)
카드에서 뺐으니 두 길 중 선택:
1. **폐기** — veil_comment 하나로 통일 (가장 단순).
2. **역할 분리** — `description` = 순수 환경/분위기(위협 언급 금지), `veil_comment` = VEIL의 전술·정서 읽기. 이러면 카드에 둘 다 띄워도 안 겹친다.
   - 현재 둘이 겹치는 대표 예: `rooftops` desc "탁 트여 노출된다" / veil "시야 트이지만 노출돼요" (거의 동일).

## 5. 참고 — 현재 게임 내 두려움 관련 텍스트 전량 (워딩 다시 쓸 때 기준)

**일반 모드 `BRIEFINGS`** (7스테이지):
- s2: "중간이에요. 저격수 자리가 늘어요. 시야 끊기는 곳 골라요." / "경비가 예상보다 많아요. 제 판단이 틀렸어요." / "저 문 너머는… 잠깐. 아니에요. 가던 길 가요."
- s4: "여기서부터 제 시야가 늦어요. 머리 위는 직접 봐줘요." / "잠깐… 아니에요. 보여요. 가요." / "이 임무 의뢰인이 누군지 알아요? 저도 몰라요. 그냥 궁금했어요."
- s6: "이상하죠. 다 와서 더 안 보여요." / "끝까지 같이 가요. 제가 보는 한." / "마지막이에요. …저도 긴장돼요."

**스토리 모드 `STORY_BRIEFINGS`** (5스테이지):
- s2: "이 구역은 오래됐어요. 오래 닫혀 있었고요." / "저 문 너머는… 잠깐. 아니에요. 가던 길 가요."
- s3: "이상하죠. 다 와서 더 안 보여요." / "끝까지 같이 가요. 제가 보는 한."

→ 보다시피 각 stage 풀에 **두려움 줄과 비-두려움 줄이 공존**한다. 이게 §1-1 희석의 실체다.
