# VEIL 대사 어투 아크 — 현재 대사 + 재작성 제안 (검토용)

> ⚠️ **대체됨 + 구현 완료(2026-06-13).** 어투 아크를 **ACT(진행도)** 에 묶던 이 문서는 → **신뢰도+사망**에
> 묶는 [`veil_trust_arc.md`](veil_trust_arc.md) + 대사 grid [`veil_pool_remap.md`](veil_pool_remap.md)로
> 대체·구현됐다. 여기 ACT1/2/3 풀은 COLD/THAW/WARM 밴드로 재활용됨(인덱싱 축만 stage→신뢰). 이 문서는
> **이력 보존용**(설계 경과). 라이브 대사는 `VeilDialogue.gd` 참조.

> **목적:** VEIL의 어투를 캐릭터 아크로 만든다. 초반엔 차가운 격식체(`~습니다/~십시오`)로 거리를
> 두다가, 후반으로 갈수록 친근한 `~해요체`로 풀린다. "함께 보낸 시간"이 신뢰/의존으로 익는 과정을
> *말투 자체*로 보여준다(= 신뢰/엔딩 테마와 직결).
>
> **검토 방법:** 각 표의 **현재** → **제안**을 보고, 채택/수정/반려를 표시해 주세요. 확정된 것만
> 코드에 반영합니다. (이 문서는 리뷰용 — 아직 코드 미반영. 단 튜토리얼은 이미 ACT1 격식체로 적용됨.)

## 어투 아크 매핑

| 구간 | 스테이지(일반/스토리) | 어투 | 톤 |
|---|---|---|---|
| **ACT1** | 0~1 / 0~1 | `~습니다 / ~십시오`, "요원" | 차갑고 사무적인 관제사. 거리감. `저도`(VEIL의 약함) 아직 없음 |
| **ACT2** | 2~4 / 2 | 격식 ↔ `~해요` **혼용**, `저도` 등장 | 균열 시작. 격식 끝에 사적인 말이 새어나옴 |
| **ACT3** | 5~6 / 3~4 | `~해요체` | 친근·걱정·사과·농담. (현재 대부분 이미 이 톤) |

**핵심 관찰:** 현재는 ACT1부터 전부 `~해요체`라 "따뜻해지는 변화"가 안 느껴진다. **ACT1을 격식체로
내리는 것이 이 작업의 본질**이다. ACT3은 대체로 유지, ACT2는 둘을 잇는 전환.

**원칙:** ① 같은 종결어미 연속 반복 금지(특히 `~게요 ~게요`) ② 40자 내외 ③ "요원" 호칭 유지
④ 격식체에선 `저도/미안해요` 같은 VEIL의 취약함을 빼고, ACT2부터 새어나오게.

## 1. 진입 인트로 (VeilDialogue.INTRO_VEIL) — ACT1

| 현재 | 제안(격식체) |
|---|---|
| ...연결됐어요. 들리세요, 요원?\n이 안은 도면이 없어요. 제가 보이는 대로 알려줄게요.\n멀리는 제가 봐줄 테니, 눈앞은 요원이 맡아줘요.\n외곽부터, 천천히 가요. | ...통신 연결됐습니다. 들립니까, 요원?\n이 안은 도면이 없습니다. 보이는 대로 전달하겠습니다.\n멀리는 제가 봅니다. 눈앞은 요원이 맡으십시오.\n외곽부터, 천천히 진입합니다. |

## 5. 레벨업 추천 (VeilDialogue) — ACT 무관(시스템 안내), 톤 중립 유지 권장

상성/태그 추천은 게임 기능 안내라 ACT 톤보다 **명료함**이 우선. 현재 `~해요`체 유지를 권장하되,
ACT1에서만 약간 격식을 섞는 변형도 가능(검토 필요). 어미 반복만 점검.

현재(유지 권장): 방패병이 정면을 막아요. 폭발물이면 방패째 뚫어요. / 저격수가 노려요. 글라이드로 떠서
사선을 흔들어요. / 드론이 위에서 와요. 연사로 쓸어내요. / 폭격기가 붙기 전에. 화력을 올려두는 게 좋아요.
/ (제네릭) 이번엔 어느 쪽도 나쁘지 않아요. 요원이 골라요. 등 — **전부 유지**.

## 6. 위협 호출 (VeilSight) — 전역, ACT 변형 검토 필요

이 줄들은 게임 내내 쓰이는 짧은 호출이라 단일 문자열로는 ACT를 못 가린다. **선택지:**
ⓐ ACT1 격식 버전 하나로 통일(첫 등장이 초반이라) / ⓑ ACT별 변형 추가(코드 분기 필요).

| 상황 | 현재 | 제안 ⓐ(격식 통일) |
|---|---|---|
| 인트로(첫 마킹) | 위험한 건 제가 먼저 봐요. 화면 끝에 짚어둘게요. | 위험 요소는 제가 먼저 포착합니다. 화면 끝에 표시하겠습니다. |
| 일반 호출 | (방위), 표시해 둘게요. | (방위), 표시하겠습니다. |
| 붕괴(degraded) | (방위) 어딘가... 저도 잘 안 보여요. 직접 살펴요. | *유지* (degraded=ACT3라 해요 적절) |

## 8. 루트 추천/진입 멘트 (RouteData.veil_comment / entry_comment)

루트는 여러 스테이지에 걸쳐 등장 — 단일 문자열이라 ACT를 정확히 못 가린다. **권장:** "전술 정보"
성격이라 **중립~약식 격식 베이스**로 두고, 아크는 stage-indexed 브리핑/사망이 운반. 단, ACT1 전용
루트(외곽/옥상)는 격식체로 내릴 수 있다.

| 루트(주 등장) | 현재 veil_comment | 제안 |
|---|---|---|
| 외곽 진입로 (ACT1) | 여기로 가요. 경비도 약하고, 길도 단순해요. | 이쪽입니다. 경비도 약하고, 경로도 단순합니다. |
| 외벽 옥상 (ACT1) | 옥상으로 갈래요? 시야는 트이지만 그만큼 노출돼요. | 옥상 경로입니다. 시야는 트이지만 그만큼 노출됩니다. |
| 폐쇄 지하철 (ACT1~2) | 옛 지하철이에요. 좁고 어두워요. 대시 써서 함정 넘어가세요. | (전환) 옛 지하철입니다. 좁고 어둡습니다. 대시로 함정을 넘으십시오. |
| 감시탑 (ACT2) | 감시탑은 위험해요. 저격이 많아요. 엄폐 짧게, 이동은 빠르게. | 감시탑은 위험합니다. 저격이 많습니다. 엄폐는 짧게, 이동은 빠르게. |
| 지하 인입로/냉각/병동 (ACT2) | (현행 해요체) | 혼용 — 검토 시 결정 |
| 데이터센터/핵심부/탈출/블랙아웃/??? (ACT3) | (현행 해요체) | *유지* |

entry_comment(진입 직후)도 동일 원칙 — 외곽/옥상만 격식 예시:
- 외곽: "외곽으로 들어왔어요. 차분히 살펴봐요." → "외곽에 진입했습니다. 차분히 살피십시오."
- 옥상: "옥상이 출구예요. 멈추면 저격에 잡혀요. 계속 움직여요." → "옥상이 출구입니다. 멈추면 저격에 노출됩니다. 계속 이동하십시오."

## 9. 신뢰도 프리픽스 (GameState.TONE_PREFIXES) — 별도 축(trust), 보류

추천 멘트 앞에 붙는 prefix는 **trust 점수 기반**(어투 아크=진행도와는 다른 축). 어투 아크를 진행도로
가면 이 둘이 충돌할 수 있으니, **프리픽스는 일단 유지**하고 아크 적용 후 재검토 권장.

현재: high `["당신이라면, ", "역시 당신이에요. ", "맞춰가볼게요. ", "믿어요. "]` / warm `["", "들어봐요, ",
"", "제 의견은요, "]` / neutral `["", "", "참고로, ", "보세요, "]` / cool `["음… ", "글쎄요. ", "흠… ",
"잘 모르겠지만, "]` — **검토 후 결정**.

## 10. 엔딩 (EndingResolver) — ACT3 종착, 전부 유지

엔딩은 아크의 종착점(가장 친근·사적)이라 **현행 `~해요체` 전부 유지**. 어미 반복만 점검:
- C 후속 "...그 잠긴 문 안쪽까지 같이 가게 되면, 그때 물어볼게요." 등 — 단일 `~게요`라 OK.

(엔딩 A/B/C/D 전문은 `EndingResolver.gd`에 있음 — 변경 없음.)

## 적용 시 주의

- 브리핑/사망은 stage-indexed라 ACT 적용이 깔끔(코드 분기 불필요, 풀만 교체).
- 위협 호출·루트 멘트·프리픽스는 단일 문자열/별도 축이라 **ACT 변형하려면 코드 분기 필요** — 위에서
  ⓐ(통일) vs ⓑ(분기)를 골라주세요.
- 확정 후 반영 순서 제안: ACT1(가장 큰 변화) → ACT2 전환 → 위협호출/루트 → 프리픽스 재검토.
