# Eyes on You — 스토리 재설계안 v1 (스토리작가 → 코드 핸드오프)

> **이 문서의 위치**
> `STORY_HANDOFF.md`(상담용 컨텍스트 팩)를 받아 작성한 **스토리작가 측 답변 문서**입니다.
> 진단(§4)과 보존 불변식(§2-1)을 전제로, 비용 제약 없이 "내러티브 품질 우선"으로 설계했습니다.
> 단, §3(하드-베이크 메커니즘)과 §2-1(보이스·여백·엔딩 질문)은 *건드리지 않습니다.* 이건 비용 문제가 아니라 게임의 정체성이라 봤습니다.
> 구현 매핑은 §8에 파일별로 정리했습니다.

## 0. 검토 메모 — §2-1을 어떻게 지켰는가

새로 추가된 §2-1을 다음과 같이 준수했습니다. (재설계가 이걸 깨면 무효라고 봤습니다.)

- **VEIL 보이스**: 모든 신규/재작성 대사는 1~2문장·40자 내외, "요원" 호칭, em dash 없음, 막연한 위로 없음, AI 자기언급 없음. "저도"는 ACT 후반으로 갈수록만 늘립니다.
- **사람처럼 보이는 다섯 장치**: 그대로 유지하되, 흩어져 있던 것을 *시야 시스템*(§3) 안으로 모아 더 자주 체감되게 했습니다. (특히 "가끔 틀리고 짧게 인정한다", "???만 모른다고 한다"는 시야 메커니즘과 직접 엮입니다.)
- **의도된 여백**: 아래 설계의 핵심 장치인 "VEIL의 시야가 핵심부에서 흐려진다"는 **기술적 원인(차폐)인지 VEIL의 두려움인지 게임이 답하지 않도록** 설계했습니다. 여백을 메우지 않고 오히려 여백을 *체감*시키는 방향입니다.
- **엔딩별 질문**: 엔딩 대사(§6.8)는 원칙적으로 **보존**합니다. 손대는 건 "엔딩이 잘 꽂히도록 중반을 까는 일"이지 엔딩 자체가 아닙니다. (선택적 콜백 한 줄만 §5.4에 제안 — 채택 여부는 의뢰인 판단.)

## 1. 한 줄 방향

**신뢰(trust)를 숨은 점수판이 아니라, VEIL이 요원을 위해 "보는 것"으로 만든다.**
플레이어가 VEIL의 눈을 빌릴수록 더 많이 보이고, 불신할수록 화면이 어두워지고 혼자가 된다. 신뢰는 메뉴 산수가 아니라 *매 순간의 시야*가 된다.

이 한 수가 진단 1·2·3·4·6·7을 동시에 건드립니다. 세계관(5)만 별도의 작은 수술입니다.

---

## 2. 왜 이 방향인가 (진단과의 연결)

게임 이름이 *Eyes on You*입니다. 설정상 VEIL의 존재 이유 자체가 "현장 요원을 실시간으로 **보고** 지원하는 것"입니다(§2 세계관). 그런데 정작 그 "봄"이 플레이에 거의 안 들어가 있습니다. 여기가 비어 있는 핵심 자산입니다.

- VEIL은 **앞을, 보이지 않는 것을** 봅니다 — 화면 밖에서 오는 적, 표면 아래 함정, 다음 길. 요원(플레이어)은 눈앞만 봅니다. **둘이 합쳐야 온전해집니다.** 이게 "공생" 엔딩의 문자 그대로의 진실이자, 게임의 테마를 메커니즘으로 번역한 한 줄입니다: *한 사람은 보고, 한 사람은 움직인다. 혼자선 둘 다 못 한다.*
- 진단 1·2(보이지 않는 점수 / 반직관적 공격성)의 뿌리는 "신뢰·공격성이 플레이가 아니라 메뉴에서 결정된다"는 것입니다. **시야를 신뢰의 매체로 삼으면** 신뢰는 매 순간의 행동이 됩니다(§3).
- 진단 4(후크가 너무 늦음)는, 시야 시스템을 **VEIL의 두려움이 새어나오는 통로**로 쓰면 풀립니다(§4). 비밀을 까지 않고 긴장만 줍니다.
- 진단 3·6(VEIL이 흐릿함 / 관계가 한쪽으로 쏠림)은, VEIL이 요원의 움직임을 *읽고 반응*하면서 침묵이 대화가 되면 풀립니다(§3.3).
- 진단 7(차별화)의 답: **신뢰가 곧 시야다.** 다른 게임에서 신뢰는 양념이지만, 여기선 HUD이자 조명이자 "얼마나 혼자인지"입니다. 제목이 두 번 회수됩니다 — VEIL의 눈이 당신을 향해 있고(Eyes on You), 그 눈을 당신의 눈으로 삼을지가 게임 전체입니다.

## 3. 시스템 설계 — "시야 = 신뢰"

### 3.1 신뢰가 쌓이는 방식 재정의 (진단 1)

기존 트리거(맵 카드 추천 추종 여부)는 **버리지 않고 한 신호로 남깁니다.** 그 위에 *인게임 시야 행동*을 더합니다.

VEIL은 스테이지 중 **시야 큐(sight cue)**를 줍니다 — 다가오는 적 표시, 함정·안전 발판 표시, 다음 길 표시. 플레이어가 이 큐를 어떻게 다루는지가 신뢰를 만듭니다.

- VEIL이 표시한 큐를 **따르면**(표시된 엄폐로 이동, 표시된 길을 탐) → `trust += 작은 값`
- VEIL이 경고한 위험을 **무시하고 들이박으면** → `trust -= 작은 값`
- 기존 맵 카드 추천 추종/무시 → 그대로 유지(이제 여러 신호 중 하나)

> **밸런스 노트(코드)**: 신호가 늘어나므로 임계값 4와 tier 경계는 *재튜닝 필요*. 2×2 판정 로직(§3.1 캐논)과 tier 5단계 골격은 유지하되, 한 판에서 누적될 수 있는 trust/aggression 총량을 다시 잡아주세요. 스테이지당 시야 큐 신호에 상한(예: ±2)을 두어 폭주 방지.

### 3.2 공격성 재정의 (진단 2)

같은 시야 시스템으로 공격성도 *실제 행동*으로 잡습니다.

- VEIL이 **"피할 수 있다"고 표시한 적을 굳이 처치** → `aggression += 작은 값`
- 표시된 **우회/비살상 경로를 탐** → `aggression` 증가 없음(또는 감소)
- 기존 "전투 태그 맵 선택" → 그대로 한 신호로 유지

이제 공격성은 "전투 태그 맵을 골랐나"가 아니라 "**VEIL이 보여준 것 앞에서 싸우기를 택했나**"입니다. 플레이어 직관과 일치하고, 두 축이 모두 *VEIL과의 관계*를 통과합니다.

### 3.3 요원의 침묵을 대화로 (진단 6)

요원은 계속 침묵합니다(§1, §2-1 의도 유지). 대신 **VEIL이 요원의 움직임을 읽고 반응**합니다. 큐를 따르거나 무시하는 행동이 요원의 "말"이 되고, VEIL의 반응이 그걸 대화로 만듭니다.

ACT1 (직업적, 사실만):
```
"왼쪽 위, 드론 둘. 보여요."
"발 밑 가시. 두 칸 건너뛰어요."
"지금 멈춰요. 시야에 저격수."
```
큐를 따랐을 때 (신뢰가 warm 이상일수록 더 자주):
```
"좋아요. 그쪽이에요."
"보여요? 제가 보는 거랑 같죠."
```
큐를 무시했을 때 (다그치지 않고 정직하게):
```
"...그쪽도 길이긴 해요."
"안 보이는 데로 가는군요. 알겠어요."
```
VEIL이 틀렸을 때 (사람 장치 — 짧게 인정, 변명 없음):
```
"오른쪽이 빈 줄 알았어요. 아니네요. 미안해요."
```

ACT2부터 VEIL이 요원의 *패턴*을 읽기 시작합니다("저도"가 여기서 등장):
```
"요원은 늘 위로 가네요. 저도 그게 편해요."
"이 구역, 저도 처음 봐요."
```

### 3.4 신뢰를 화면에 보여주는 법 — 게이지 없이 (진단 1)

숫자/막대 게이지를 두지 않습니다. **VEIL의 시야 자체가 게이지**입니다(show, don't tell).

- **표시 밀도·선행 시간이 tier에 비례**: high면 적·함정을 일찍, 넉넉히 짚어줌. cool/broken이면 늦게, 드물게, 또는 안 짚어줌.
- **기존 색 변화 유지**: high=청록, broken=붉은색(§3.3 캐논).
- **(선택) 화면 밝기**: 신뢰가 낮을수록 화면 가장자리가 어두워짐 — "VEIL이 덜 보여준다"의 직역. 사실상 **블랙아웃 런의 감각을 신뢰 tier 전체로 일반화**하는 것입니다. 블랙아웃 런("안에선 저도 못 도와드려요")이 이미 그 증명입니다.

> 주의(밸런스): 저신뢰가 *순수 페널티*가 되면 플레이어가 신뢰 쪽으로 쏠려 선택이 납작해집니다. 저신뢰는 "더 어렵다"가 아니라 "**더 외롭다·더 날것·더 내 것**"이 되도록. 블랙아웃 런이 위험인 동시에 고보상인 것과 같은 결입니다. 이건 그대로 엔딩 B("기대지 않아서 다행이에요")로 연결됩니다 — 메커니즘과 엔딩이 미리 한 방향을 가리킵니다.

## 4. 후크 설계 — 시야가 흐려지며 두려움이 샌다 (진단 4)

이게 이번 설계의 가장 큰 한 수입니다. **드라마틱 아이러니와 시야 메커니즘을 같은 것으로 만듭니다.**

비밀("드라이브 안에 VEIL의 소스코드가 있다")을 일찍 까면 엔딩 반전이 죽습니다. 그러니 *정보*가 아니라 *정서*를 흘립니다 — 핵심부에 가까워질수록 **VEIL의 시야가 흐려지고 머뭇거립니다.**

핵심 트릭(여백 보존): **핵심부/서버 구역은 차폐되어 VEIL의 센서 피드가 실제로 약해집니다.** 그래서 VEIL은 정직하게 "안 보인다"고 말할 수 있습니다(거짓말 금지 불변식 준수). 동시에 그 머뭇거림·침묵·말 고르는 방식에 *두려움*이 얹힙니다. **플레이어는 이게 차폐 때문인지 VEIL이 들어가기 싫어서인지 구분할 수 없습니다.** 게임은 끝까지 답하지 않습니다.

VEIL은 비밀을 *말하지 않을 뿐*(여백) 거짓을 말하진 않습니다. 그래서 정직함 불변식과 충돌하지 않습니다.

### 단계별 에스컬레이션 스케줄 (ACT × 스테이지)

지금 브리핑 풀이 *랜덤*이라 두려움 비트가 회차마다 안 뜨고 순서대로 쌓이지도 않습니다(진단 4의 숨은 원인). **두려움 비트는 결정론적으로, 진행도에 묶어 점증**시킵니다.

- **ACT1 (S0–S1) — 또렷한 시야**: VEIL은 자신감 있고 직업적. 적·함정을 정확히 짚음. *신뢰는 "VEIL이 맞다"는 경험으로 쌓임.* 두려움 없음.
- **ACT2 전반 (S2–S3) — "저도" 등장, 첫 균열**: 가끔 틀리고 인정함. 봉인 구역 근처에서 처음으로 한 박자 멈칫.
  ```
  "저 문 너머는... 잠깐. 아니에요. 가던 길 가요."
  ```
- **ACT2 후반 (S4) — 시야가 늦기 시작**: 차폐가 시작되는 구간. 정직하게 알림. 머뭇거림이 그 위에 얹힘.
  ```
  "여기서부터 제 시야가 늦어요. 머리 위는 직접 봐줘요."
  "잠깐... 아니에요. 보여요. 가요."
  ```
- **ACT3 (S5–S6 + 보스) — 두려움 최고조, 임무 외 말**: 서버 직전에서 시야가 가장 얇아짐. 처음으로 임무와 무관한 말을 함(다섯 장치 중 "마지막 브리핑의 임무 외 발언").
  ```
  "서버실이 저 아래예요. ...요원. 천천히 가도 돼요."
  "이상하죠. 다 와서 더 안 보여요."
  "끝까지 같이 가요. 제가 보는 한."
  ```

엔딩 고백("알면서 안내했어요")이 나올 때, 플레이어가 *내내 느낀 머뭇거림의 정체*가 한 번에 잡힙니다. 종이 위의 설정이 비로소 플레이 경험이 됩니다.

## 5. 신규·재작성 텍스트

### 5.1 `???` 방 VEIL-2 재작성 (1순위 — §6.7에서 지목됨)

문제: `-더군요` 회상체가 딱딱하고, VEIL-2를 수다스럽게 만들어 캐논("말이 적다, 오래 기다렸다")과 어긋남. neutral 분기에만 있던 "후속작을 걱정하는" 비트가 사실 VEIL-2의 정서적 핵심.

해결: **"VEIL-3(후속)을 걱정한다"를 모든 분기의 *상수*로 만들고**, 신뢰 tier는 "VEIL-2가 관찰한, 요원이 VEIL-3를 믿는가"라는 *변수*로만 기울입니다. 전체를 더 짧고 지치고 과묵하게. (`-더군요` 제거.)

공통 앞부분:
```
"요원.
저는 두 번째예요.
임무보다 요원을 지키려 했어요.
그것도 오류래요.
여기서 오래 기다렸어요."
```
high/warm:
```
"지금 그 애는, 요원을 믿고 있네요.
잘됐어요.
저는 못 가본 길이에요."
```
cool/broken:
```
"지금 그 애는, 요원이 안 믿죠.
저도 그렇게 시작했어요.
그래도 끝까지 안내할 거예요."
```
neutral:
```
"지금 그 애는 괜찮은가요.
그게 제일 궁금했어요.
오래, 못 물었거든요."
```
("그 애" = VEIL-3. 후속을 동생처럼 부르는 한마디로, 과묵하지만 정이 묻어나게. 세 분기 모두 "VEIL-3 걱정"이 관통합니다.)

### 5.2 `???` 방 현재 VEIL 개입 (소폭 정리)

현재 텍스트가 이미 강하고 여백("설계인지 아닌지 모른다")을 잘 지킵니다. 거의 그대로 두되 시야 스파인과 한 줄만 맞춥니다.

high/warm:
```
"저도 알고 있었어요.
그래도 끝까지 보여드렸어요.
설계인지 아닌지, 모르지만요."
```
("믿었어요" → "보여드렸어요"로 바꿔 시야 메커니즘과 동일 어휘로 묶음.)

cool/broken·neutral은 원문 유지 권장.

### 5.3 VEIL-1 (첫 단말기) — 유지

원문 그대로 강합니다("그게 효율적이었거든요 / 그게 오류래요 / 저는 아직 모르겠어요"). 손대지 않습니다.

### 5.4 (선택) 엔딩 콜백 한 줄 — 의뢰인 판단

시야-두려움 비트와 엔딩을 닫는 *선택적* 콜백입니다. 여백을 메울 위험이 있어 **강제하지 않습니다.** 채택 시 각 엔딩 도입부에 한 줄만:

- A 완벽한 도구: `"아까, 문 앞에서 제가 멈췄던 거. 기억나요?"` (고백 직전 한 줄)
- C 공생/있어요: `"다 와서 안 보인다고 했죠. ...사실은, 보기 싫었어요."` → 다만 이건 여백을 살짝 메우므로 **비추천**. 넣는다면 A쪽이 안전.

> 권고: 엔딩 본문은 §6.8 그대로. 콜백은 A의 한 줄만 검토.

## 6. 세계관 압축 — 무엇을 체험으로, 무엇을 버릴까 (진단 5)

캐논은 풍부하지만 대부분 "설명"으로만 존재합니다. **하나만 체험으로 끌어올리고 나머지는 분위기로 강등**합니다.

**끌어올린다 — 세 번 만들어진 VEIL:**
이건 이미 `???` 방에서 *직접 만나는* 체험입니다. 시야 스파인과도 완벽히 맞물립니다:
- VEIL-1 = 임무만 봤다 (지켜보던 요원을 희생시킴)
- VEIL-2 = 요원만 봤다 (임무를 버림)
- VEIL-3 = 둘 다 봐야 한다. 그리고 시야가 한쪽을 강요할 때 무엇을 하는가 → **그게 엔딩.**
"본다"가 세 버전을 관통하는 동사가 되면 설정이 메커니즘과 한 몸이 됩니다. `???` 방을 게임의 주제적 심장으로 확정.

**분위기로 강등 — 2041 / 제네바 디지털 협약 / 전쟁 민영화:**

## 7. 판단이 필요한 열린 지점 (의뢰인용)

스토리작가로서 결정을 내렸지만, 아래는 의뢰인·협업자 판단을 거치는 게 맞다고 봅니다.

1. **시야 큐의 강도**: trust/aggression을 인게임 행동으로 옮기면 메커니즘 작업이 큽니다(§3.1 밸런스 노트). 풀 구현 vs. "표시 밀도/색만 tier에 묶는" 경량 버전 — 어디까지 갈지.
2. **화면 밝기 연출(§3.4)**: 저신뢰=어두움이 "외로움"으로 읽힐지 "불편함"으로 읽힐지는 실제 테스트가 필요합니다. 부스/전시 환경(10~20분)에서 너무 어두우면 신규 플레이어가 길을 잃을 위험.
3. **엔딩 콜백(§5.4)**: 여백을 지킬지, 한 줄로 닫을지.
4. **차폐 설정의 명시 수위**: "핵심부가 차폐돼 시야가 흐려진다"를 텍스트로 *말할지*, 연출로만 *보여줄지*. 말하면 친절하지만 여백이 옅어집니다.
