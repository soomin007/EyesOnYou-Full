# Eyes on You — PRD (제품 요구사항)

> **제품 결정·비목표·성공 기준·UX 원칙·리스크의 단일 소스.** 게임 내용·조작은 [`README.md`](README.md),
> 구현 사양·시스템 수치는 [`docs/SPEC.md`](docs/SPEC.md), 진행 상태·다음 작업은 [`docs/design/backlog.md`](docs/design/backlog.md)를 본다.

## 1. 제품 한 줄 정의

> **"AI 파트너 VEIL과 함께 7개 임무를 클리어하는 횡스크롤 로그라이트. 마지막에 VEIL이 누구였는지 드러난다."**

근미래 보안기업 현장 요원이 상황실 AI **VEIL**의 조언을 들으며(또는 무시하며) 짧은 횡스크롤 스테이지를
클리어한다. 누적된 선택(추천 수용률 × 전투/우회 비율)이 4종 결말 중 하나를 결정한다.

## 2. 타깃 / 배포

| 항목 | 내용 |
|---|---|
| 플랫폼 | 브라우저(Godot 4.6 Web Export) — GitHub Pages 자동 배포 + 데스크톱 |
| 접근 | 전시·부스에서 QR/링크로 즉시 플레이 ("들어와 한 번 깨고 나가는" 구조) |
| 1회 플레이 | 8~15분 |
| 기기 | 데스크톱 브라우저, 키보드 또는 Xbox 호환 게임패드 |
| 외부 의존성 | 없음 (API/서버/계정 없음) |

→ 첫 30초 안에 조작이 손에 붙고, 8~15분 안에 완결되어야 한다.

## 3. 핵심 가치 제안 (Why this game)

1. **짧고 완결된 서사** — 한 번의 플레이로 끝나되, 결말 4종이라 다회차 동기가 있다.
2. **묻지 않은 질문에 게임이 답한다** — 플레이어는 VEIL에 대해 묻지 않는다. 마지막 컷에서 게임이
   그 질문을 대신 던진다(결말 C에서만 직접 묻는 선택지가 열림).
3. **가벼워 보이지만 무거운 선택** — 루트·스킬 선택은 즉각적 게임플레이 결정처럼 보이나, 실제론
   결말을 가르는 두 축을 누적시킨다.

## 4. 결말 = 제품의 핵심 페이오프

두 축(추천 수용률 × 공격성)이 직교해 4종 결말로 갈린다 — **A 완벽한 도구 / B 혼자였던 사람 /
C 공생 / D 유령 임무**. 두 축이 *서로 독립*이어야 한다는 것이 제품 요구사항(조언을 따름 ≠ 전투 선호).
정확한 조건·대사·점수 산식은 [`docs/STORY.md`](docs/STORY.md)·[`docs/design/veil_trust_arc.md`](docs/design/veil_trust_arc.md), 구현은 `EndingResolver.gd`.

## 5. VEIL 페르소나 — 제품 정체성

- 1~2문장, 침착·간결, 끝에 "요원". **가끔 판단이 틀린다** — 늘 따르는 게 정답이 되면 안 된다(결말 다양성의 핵심 장치).
- 대사는 전부 하드코딩(LLM/외부 API 금지). 톤 가이드 상세는 [`docs/STORY.md`](docs/STORY.md) Part I.

## 6. UX 원칙

1. **튜토리얼 화면 없음** — 첫 브리핑이 곧 튜토리얼. 키 가이드는 화면 모서리 상시 노출.
2. **타자기 효과** — VEIL 대사는 글자 단위로 흐른다. 스킵 가능.
3. **결말 D의 정적은 의도** — 스킵 불가. 그게 그 결말의 본문이다.
4. **사망 페널티 최소** — 같은 스테이지 시작점 재개, 스코어 누적 유지.
5. **모든 선택이 보이게** — 위험/보상은 ●, 숨은 루트만 `?`. 미니멀 벡터 UI.

## 7. 비목표 (Non-goals)

- 세이브/로드 없음 (8~15분 단발 플레이 전제)
- 다국어 없음 (한국어 단일)
- 온라인 / 리더보드 / 계정 없음
- 모바일 터치 컨트롤 없음 (데스크톱 키보드/패드 전제)
- 챕터 / 시즌 / DLC 없음 (한 번의 7스테이지로 완결)

## 8. 성공 기준

| 지표 | 목표 |
|---|---|
| 1회 플레이 완료율(시작→결말 도달) | 부스 환경 60% 이상 |
| 평균 1회 플레이 시간 | 8~15분 |
| 결말 분포 | 4종 모두 도달 가능(한 결말로 80%↑ 쏠림 없음) |
| Web 빌드 첫 로딩 | 30초 이내 |
| 한글 렌더 결함 | 0건 |

## 9. 리스크 / 오픈 질문

| 항목 | 리스크 | 대응 |
|---|---|---|
| 결말 D의 정적 | "버그인가?" 오해 | 미세 노이즈/페이드로 "의도된 정적" 암시 |
| VEIL 조언이 항상 옳다는 학습 | 결말 다양성 붕괴 | 의도적으로 빗나간 조언을 풀에 박음 |
| 부스에서 결말까지 못 보고 떠남 | 핵심 페이오프 미체험 | 스테이지를 화면 3~4개 분량으로 짧게 |
| 초보자 난이도 체감 미상 | 부스 이탈 | 플레이 피드백 설문으로 수집(→ backlog) |
