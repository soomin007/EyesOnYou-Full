## 0. 응답 요약 + 핵심 원리

진단 §1 전부 동의합니다(풀 희석·단일 채널·보고체 워딩·짧은 스토리 모드 곡선). 그런데 §1-3("전술 코멘트로 읽힌다")이 *진짜 뿌리*이고, 나머지는 그걸 증폭한 것입니다. 그래서 워딩을 고치기 전에 **무엇이 무서움인지**를 다시 정의합니다.

> **두려움 = VEIL이 감정을 연기하는 게 아니라, "내가 당신을 위해 본다"가 "당신이 날 대신 봐줘요"로 뒤집히는 것.**

VEIL의 정체성·존재 이유는 *본다*입니다. 그러니 가장 무서운 건 VEIL이 "무서워요"라고 말하는 게 아니라, **늘 침착하고 정직하던 VEIL이, 자기 시야가 꺼지는 걸 침착하게·정직하게 인정하면서 요원에게 "봐달라"고 기대기 시작하는 것**입니다. 침착함이 깨지는 게 아니라, 침착함을 유지한 채 *핵심 기능이 무너지는* 걸 정직하게 보고할 수밖에 없는 상황 — 그게 공포입니다. 그리고 정직함(캐논의 핵심 후크)이 그 공포의 전달 수단이 됩니다.

이게 사용자의 "전이랑 뭐가 달라졌지?"에 답합니다. 전엔 아무것도 안 변했습니다 — 줄이 바뀌어도 VEIL은 여전히 "유능한 눈"이었으니까요. 이제는 **관계가 눈앞에서 역전**됩니다. Claude Code가 예시로 쓴 `제가 못 보면 요원이 봐줘요`가 정확히 그 역전이고, v3는 그 방향으로 전부 통일합니다.

> ⚠️ 톤 가드: Claude Code 예시 중 `무서운 말 같지만…`, `왜 이러지`는 VEIL이 *자기 감정을 라벨링/패닉*하는 쪽이라 §2-1("감정은 직접 말하지 않되 배어난다")과 충돌합니다. 더 무서운 건 VEIL이 끝까지 침착하게 "안 보인다 / 이런 적 없다 / 요원이 봐달라"를 *사실로* 말하는 것입니다. 패닉은 캐릭터를 바꾸고, 침착한 인정은 캐릭터를 무너뜨립니다. 후자가 맞습니다.

## 2. A. 두려움 풀 동질화 — 어느 줄이 뽑혀도 역전이 운반되게 ★최우선

구조(랜덤 1줄)는 유지하되, **풀 안의 모든 줄이 같은 비트**를 운반합니다(워딩만 다양). 안심 줄은 풀에서 빼서 별도 채널로(§4). 아크는 *역전의 3단계*로 설계합니다: **흠칫(감춤) → 시야 새기 시작(요원에게 넘기기) → 거의 못 봄(완전한 역전).**

### 일반 모드 `BRIEFINGS`

**s2 — 1단계 흠칫 (ACT2 시작, 미세한 균열).** 전부 "VEIL이 앞의 무언가에 흠칫하고 얼버무린다":
```
"저 문 너머는... 잠깐. 아니에요. 가던 길 가요."
"앞에 뭔가... 아니에요. 제가 잘못 봤어요. 가요."
"방금... 아니에요. 신경 쓰지 말아요. 저도 가끔 이래요."
```
(감추는 걸 *signposting*하지 않게 — 플레이어는 "뭔가 있다"는 불안만 느끼고, VEIL이 숨긴다는 건 엔딩에서야 알게 됩니다. 여백 보존.)

**s4 — 2단계 시야가 새기 시작 (차폐 시작, 요원에게 넘기기).** 전부 "VEIL의 봄이 끊기고, 요원에게 봐달라 한다":
```
"여기부터 제 눈이 잘 안 닿아요. 머리 위는 요원이 봐줘요."
"시야가... 자꾸 끊겨요. 이런 적 없는데. 요원이 직접 봐요."
"제가 못 보는 데가 생겨요. 거기는 요원 거예요."
```

**s6 — 3단계 거의 못 봄 (서버 직전, 역전 완성).** 전부 "VEIL이 거의 못 보고, 이제 요원이 자기 대신 본다":
```
"다 와서... 앞이 안 보여요. 이런 적 없어요."
"여기, 제 눈이 안 닿아요. 이제 요원이 봐줘요. 저 대신."
"지금부터 제가 틀릴 수도 있어요. 미리 말해둘게요."
```
(`저 대신` / `미리 말해둘게요`가 핵심 — 패닉 없이, 침착한 정직함이 오히려 서늘하게. `제가 틀릴 수도 있어요`는 캐논 장치 "조언이 가끔 틀리고 인정한다"를 *선제적으로* 켜서 더 불길합니다.)

### 스토리 모드 `STORY_BRIEFINGS` (5스테이지 — 더 진하게)

곡선이 짧으니(s2 흠칫 → s3 최고조 단 2비트) 비트를 더 또렷하게. 1·2단계를 s2가 겸하고, s3가 역전 완성을 확실히 칩니다.

**s2 — 흠칫 + 시야 새기 시작(겸함):**
```
"저 문 너머는... 잠깐. 아니에요. 가던 길 가요."
"여기부터 제 눈이 잘 안 닿아요. 머리 위는 요원이 봐줘요."
```
**s3 — 역전 완성(최고조):**
```
"다 와서 앞이 안 보여요. 이런 적 없어요. 요원이 봐줘요."
"제 눈이 여기서 끊겨요. 이제 요원 차례예요. 저 대신."
```

## 3. B. "안 보인다 → 무섭다" 표면화 — 어떻게 워딩이 작동하는가

패닉이 아니라 *침착함의 균열*로 갑니다. 세 가지 장치로:

1. **역전 어휘**: `요원이 봐줘요` / `요원 거예요` / `저 대신` / `이제 요원 차례예요`. 보던 존재가 보임을 넘기는 순간이 사건입니다.
2. **이상 신고**: `이런 적 없어요` / `이런 적 없는데`. 늘 보던 VEIL이 "처음 있는 일"이라 말하는 것 = 세계가 어긋났다는 신호.
3. **호흡(em dash 대체)**: `…`와 끊긴 짧은 문장으로 머뭇거림. `다 와서... 앞이 안 보여요.` 의 `...`가 정서를 만듭니다. 보고체(`시야가 늦어요`)엔 이 호흡이 없어서 담담했습니다.

금지선 재확인: `무서워요` 직접 진술 ✕, `왜 이러지` 패닉 ✕, 막연한 위로 ✕, AI 자기언급 ✕. VEIL은 끝까지 침착하고 정직합니다 — 그래서 더 무섭습니다.

## 4. C. 채널 분산 — 단일 브리핑 의존을 깬다 (결정)

§1-2가 맞습니다. 누적이 핵심인 연출을 *스킵되는 단일 화면*에만 실으면 누적이 성립 안 합니다. 채널을 다음처럼 역할 분담합니다. **연결은 Claude Code가, 텍스트는 아래 제공.**

| 채널 | 운반 비트 | 이유 |
|---|---|---|
| 브리핑(`BRIEFINGS`) | **상승**(흠칫→새기 시작) | 셋업. 스킵돼도 누적의 일부. |
| **인게임 자막(`entry_comment`/위치 트리거, ACT3 결정론)** ★ | **최고조(역전 완성)** | 플레이 *중*에 떠서 스킵 불가. **이게 §1-2의 실제 해결.** |
| 사망 멘트(ACT3) | 역전 보강 1줄 | 이미 개인적 톤. 죽어도 아크가 들림. |
| 보스 처치 후 | 안심 줄 *이전(以前)* | 풀에서 뺀 `제가 보는 한`의 새 집. |

**★ 인게임 자막 (최고조 — 가장 중요).** 핵심부/서버 접근 시 *결정론적으로* 뜨는 자막. 기존 위치 트리거 자막(§6.6 방식)과 동일하게 구현. ACT3에서만:
```
(서버 접근 자막) "여기는... 제가 안 보여요. 요원이 봐줘요. 저는 들을게요."
(핵심부 진입 자막) "제 눈이 여기서 멈춰요. 이제 요원 거예요."
```
스토리 모드는 비트가 둘뿐이라 이 인게임 자막이 *특히* 중요합니다 — s3 인게임에 반드시 하나 박아주세요.

**사망 멘트(ACT3 추가 1줄)** — 시야 역전과 죽음을 묶음:
```
"제가 잘 못 봐서… 미안해요. 다시 가요."
```
(VEIL이 자기 실패로 죽음을 받아안음 = 정직함 장치 + 역전 아크 보강. `미안해요`는 캐논 허용.)

**보스 처치 후(안심 줄 재배치).** 풀에서 뺀 `끝까지 같이 가요. 제가 보는 한.`을 여기로. 보스를 막 넘고 서버가 코앞일 때 이 다짐이 나오고, **엔딩에서 시야가 완전히 끊기며** 그 다짐이 회수됩니다(`제가 보는 한` → 보는 게 끝나는 순간). 기존 보스 처치 후 자막에 한 줄 추가:
```
"처리됐어요, 요원. / 서버실이 바로 앞이에요. / 끝까지 같이 가요. 제가 보는 한."
```

**(선택) 보스전 중간 한마디** — 최고조에서의 역전:
```
"여기선 제가 거의 못 봐요. 요원을 믿을게요."
```
클라이맥스에서 VEIL이 요원의 눈을 믿는 한마디. 강하지만 선택. 과하면 생략.

## 5. D. 맵 `description` 역할 — 카드에서 폐기 권장 (옵션 1)

**권장: 1) 폐기 (veil_comment 한 목소리로 통일).** 근거 셋:
- 사용자가 중복에 직접 불쾌감을 표했습니다("너무 겹친다"). 되살리면 같은 불만을 재생산할 위험.
- 부스/전시는 빠르게 읽는 환경 — 카드에 두 블록은 인지 부하.
- **테마적으로도 맞습니다.** 이 게임에서 세계는 VEIL이라는 *렌즈*를 통해 옵니다(Eyes on You). 카드에 중립 서술 + VEIL 목소리가 따로 있으면 "객관 세계 + 주관 VEIL"로 분리되는데, 사실 이 게임의 매력은 *VEIL의 눈을 통해서만 세계가 보인다*는 것입니다. 한 목소리가 오히려 컨셉에 충실합니다.

단, `description` 텍스트 자체는 **버리지 말고 보관**해주세요 — 나중에 순수 환경 자막(비-VEIL 분위기 레이어)이 필요해지면 재료로 씁니다.

> 만약 그래도 분위기 레이어를 원하면(옵션 2), 규칙은 **`description` = 장소가 *무엇인지*(정적·감각, 위협 언급 금지) / `veil_comment` = VEIL이 그걸 *어떻게 보고 느끼는지*(동적·관계)**. 겹침의 대표 예 `rooftops`를 이 규칙으로 고치면:
> - `description`: `바람이 분다. 발밑으로 도시가 멀다.` (환경만)
> - `veil_comment`: `옥상이에요. 시야는 트이지만 그만큼 노출돼요. 멈추지 말아요.` (VEIL의 읽기)
> 이렇게 *역할이 갈리면* 둘을 띄워도 안 겹칩니다. 하지만 위 이유로 **옵션 1을 먼저 권합니다.**

## 6. 보류 동의 + 다음 플레이테스트에서 볼 것

§4(시야=신뢰 파일럿·엔딩 콜백) 보류 동의합니다. 두려움 아크가 서야 그 위에 올립니다.

**다음 테스트에서 사용자에게 물어볼 것** — 이번엔 "어디서 무서웠나"가 아니라:
> "후반으로 갈수록 VEIL이 점점 *너한테 기대는* 느낌이 들었나? VEIL이 '내가 못 보니 네가 봐달라'고 하는 순간이 있었나?"

여기에 "그렇다"가 나오면 역전이 전달된 것이고, 두려움은 그 역전의 그림자로 따라옵니다. 여기서 안 걸리면 워딩이 아니라 *채널*(§4 인게임 자막)이 아직 약한 것이니, 최고조 비트를 브리핑에서 인게임 자막으로 더 옮기면 됩니다.
