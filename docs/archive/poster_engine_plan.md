## 현재 상태
- **엔진 포스터 소스:** `scripts/PosterCanvas.gd` (1240×1754 세로). `scripts/Poster.gd`가 SubViewport로 렌더.
- **렌더 실행:** `godot --path . res://scenes/poster.tscn --gen` → `res://poster_out/eyes_on_you_poster.png` 저장 후 종료.
  **⚠️ 헤드리스는 빈 이미지** — 반드시 **창모드(--gen)**로 사용자가 직접 돌려야 함. Claude는 렌더 결과를 못 봄 →
  사용자가 PNG를 보여주거나 Read로 확인하며 반복.
- 현재 `Poster.gd`는 **PNG만** 저장. **PDF·썸네일 미생성.**
- 현재 PosterCanvas 구성: 헤더 / VEIL 눈 / EYES ON YOU / 태그라인 / 로그라인 / 스샷 3(맵선택·스킬트리·전투) /
  정보칩 / 특징 4셀 / VEIL 인용 / 푸터. **엔딩 행 없음**(챗판엔 있음).

## 목표(챗 CLAUDE판을 시각 레퍼런스로) — `poster_out/EyesOnYou_CLAUDE/eyes_on_you_poster.png`
1. **엔딩 행 추가** — "FOUR ENDINGS … A 완벽한 도구 / B 혼자였던 사람 / C 공생 / D 유령 임무 … FOLLOW × FIGHT".
   엔딩 축을 한 줄로(신뢰 수용률 × 전투 성향 → 4결말). 챗판의 핵심 우위가 이거였음.
2. **코너 브래킷 ↔ 텍스트 겹침 해소**(사용자 지적): 상단 헤더 양끝·좌우 라벨·하단 푸터가 코너 브래킷(┌┐└┘,
   `_corner_brackets`)·테두리와 안 겹치게 **안쪽 여백 ↑** 또는 브래킷을 작게/바깥으로. 가장자리 글로우/스캔라인이
   글자 위에 안 오게.
3. **레이아웃 폴리시** — 챗판 수준의 여백·정렬·균형. (현재도 깔끔하나 엔딩 행 들어가면 세로 배치 재배분 필요.)

## 출력 보강(Poster.gd) — 제출에 필요
- **150×150 정사각 썸네일 렌더 추가.** 포스터가 세로라 크롭 아님 — 별도 정사각 캔버스(VEIL 눈 + "EYES ON YOU" +
  "SURVEILLANCE ROGUELITE" 정도)로 렌더해 `poster_out/eyes_on_you_thumb_150.png` 저장. (챗 CLAUDE 썸네일 참고.)
- **인쇄 품질** — 현재 1240px는 A4 ≈150dpi라 인쇄엔 약함. **2배(2480×3508 = A4 300dpi)**로도 한 장 뽑게(상수 스케일).
- **PDF** — Godot는 PDF 직접 출력 없음. 폼은 PNG도 허용("PDF/JPEG/PNG")이라 **고해상 PNG 제출 가능**. PDF가 꼭
  필요하면 PNG→PDF는 외부 1스텝(사용자가 변환). 계획: 고해상 PNG를 기본 제출물로, PDF는 선택.
