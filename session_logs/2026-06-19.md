# 2026-06-19 세션 로그

## 요약
플레이 피드백 두 건 반영. ① 크레딧 끝 메뉴에서 ESC 스킵 시 멈춘 크레딧 본문이 메뉴 버튼과
겹치던 문제 수정. ② 냉각 시설(cooling) 레버를 증기 분출구 옆 지면에서 안전한 발판 위로 옮기고,
당겨 솟는 발판/해치 연출을 제거(XP 3은 발판 위 직접 spawn).

## 주요 변경
- `scripts/Credits.gd` — `_show_end_menu()` 진입 시 `_scroll.visible = false`로 크레딧 본문을
  숨겨 메뉴 버튼과 분리. (자동 종료 시엔 이미 화면 위로 올라가 무해, ESC 중간 호출에서만 겹쳤음)
- `scripts/Stage.gd` `_build_cooling_secret()` — 레버 (1450,548)→(1560,380 발판 위)로 이동,
  `_spawn_closed_hatch`/`_spawn_drop_platform`/`_open_hatch`/`_descend_drop_platform` 호출 제거,
  XP 3을 발판 위(1510/1560/1610, y≈350)에 직접 배치. VEIL 자막도 "위로 올라가" 문구 제거.
- `docs/design/known_issues.md` — 두 함정 기록(비밀 레버 안전 배치 / 오버레이 메뉴 배경 컨텐츠 숨김).

## 결정 사항
- cooling 레버: 사용자 선택은 "레버 유지 + 플랫폼만 제거"(비밀칸 전체 제거 아님). 레버 보상
  콘텐츠는 살리되 해저드 근접·솟는 발판 연출만 정리. back_alley(첫 레버 튜토리얼)·rooftops의
  drop_platform 퍼즐은 동선·인과가 맞아 유지 — 공용 헬퍼 4종은 죽은 코드가 아님.
- 크레딧 겹침은 `_scroll`을 숨기는 방식(페이드 아님) — ESC 스킵 의도가 "메뉴로 즉시 이동"이라
  본문이 즉시 사라지는 게 자연스럽고, 0.4s 페이드 중 겹쳐 보이는 구간을 피함.
