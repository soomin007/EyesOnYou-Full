## 주요 변경
- `scripts/TouchControls.gd`: `_update_states()`(매 프레임, ALWAYS `_process`) 추가 — player를 group으로
  폴링해 dash(`has_skill`·`dash_cd`)·skill(`explosive` 티어·`skill_cd`·`skill_charges`) 상태를 버튼 dict에
  기록. 미보유=흐리게(0.22)+비활성(입력 무시), 쿨다운=흐리게(0.40). `restrict_combat_input`이면 사격/대시/
  스킬 `vis=false`로 숨김. `_button_at`이 비활성/숨김 버튼을 건너뛰고, `_draw_button`이 dim을 알파에 곱함.
  상태 서명(sig) 변화 시에만 재그리기.
- `scripts/OrientationGuard.gd`: `_update_portrait_pause()` 추가 — stage 그룹 존재 + portrait일 때 자동
  `get_tree().paused=true`(안내 카드 뒤 피격/추락 방지), 가로 복귀 시 해제. `_portrait_paused` 플래그로
  **우리가 건 pause만 해제** → 사용자가 직접 연 일시정지 메뉴는 보존. `_refresh`(size_changed) + `_process`
  (씬 전환으로 stage 등장/이탈, 신호 누락 대비) 양쪽에서 재확인.
- `scripts/Credits.gd`: `_input`에서 화면 홀드 손가락(`_touch_fingers`) 추적 → `_process` 빨리감기 조건에
  `or not _touch_fingers.is_empty()` 추가(SPACE 홀드와 동일). 루트가 full-rect Control이라 `_unhandled_input`엔
  탭이 안 와 `_input`에서 처리(소비 안 함 — 끝 메뉴 버튼 탭 보존).
- `scripts/ArcturusDocumentOverlay.gd`: 줄 진행/닫기 조건에 `OrientationGuard.is_tap(event)` 추가. 기존
  좌클릭 분기엔 `not is_touch_device()` 가드 — emulate 마우스가 탭당 좌클릭도 합성해 2줄 스킵되는 것 방지.

## 결정 사항
- **상태 폴링을 TouchControls 안에서** — Player/Stage를 안 고치고 group 조회로 dash_cd/skill_cd/charges를
  읽어 버튼에 반영. 서명 비교로 재그리기 최소화(쿨다운은 사실상 이진 상태라 저렴).
- **`restrict_combat_input` 재사용** — ??? 방(route_hidden)·연출에서 이미 켜지는 라이브 플래그라 전투버튼
  숨김의 단일 신호로 딱 맞음(새 플래그 불필요). 켜지는 곳: `_build_hidden_archive`, 각종 시네마틱.
- **세로 자동 일시정지는 stage에서만 + 우리 pause만 해제** — 메뉴는 시간 의존 아니라 불필요(카드가 덮음).
  `_portrait_paused` 소유권 플래그로 사용자 pause 메뉴 오해제 방지(`paused` carry 함정과 동형 설계).
- **Credits는 탭-스킵 대신 홀드-빨리감기** — 자동 스크롤 특성상 SPACE 홀드의 충실한 매핑. 탭-스킵은 시청
  중 오스킵 위험 + 끝에서 메뉴 자동 등장이 이미 있음("후반이라 급하지 않음").
- 검증: 창모드 래퍼(`_touch_check.tscn`)로 4스크립트 컴파일·인스턴스화·`_update_states` 동작 확인(FileAccess
  기록). Arcturus `_process` null 크래시 1건은 하니스가 `show_doc` 없이 인스턴스화한 인공물(실게임 무관) —
  `known_issues`에 기록. 검증 후 임시 파일 삭제.
